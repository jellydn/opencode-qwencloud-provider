opencode provider

QwenCloud models for opencode

Six chat models via OpenAI-compatible API. Wan image generation and HappyHorse video generation as native opencode custom tools. JSON config for the provider, TypeScript plugin for media.

Get started View on GitHub 
// opencode.json
"provider": {
  "qwencloud": {
    "npm": "@ai-sdk/openai-compatible",
    "name": "QwenCloud",
    "options": {
      "baseURL": "https://token-plan...",
      "apiKey": "{env:QWENCLOUD_API_KEY}"
    },
    "models": { … }
  }
}

Chat provider is config-only — no build step for the models.

Install

Two steps to get started

1

Copy the config

Drop opencode.json into your global or project config.

cp opencode.json ~/.config/opencode/
2

Set your API key

Export your Token Plan key. The config interpolates it from the environment — never touches the file.

QWENCLOUD_API_KEY=sk-... opencode

Media generation

Wan images + HappyHorse video

Custom opencode tools. The AI calls them natively when you ask for image or video generation — or use the slash commands directly.

Wan image generation

Text-to-image with wan2.7-image and wan2.7-image-pro. Outputs at 1K, 2K, and 4K. Synchronous — images save to your working directory immediately.

/wan a cyberpunk cat in neon rain

HappyHorse video generation

Text-to-video, image-to-video, and reference-to-video. Async with progress polling. 3–15 second output.

/happyhorse a drone shot over the ocean

Plugin setup

Local (copy files)

cp -R plugin/ ~/.config/opencode/plugin/
cp -R command/ ~/.config/opencode/command/

npm (add to opencode.json)

"plugin": ["opencode-qwencloud-provider"]

Features

Provider + plugin in one package

Config-only chat

The chat provider is pure JSON. opencode reads it directly — streaming, tool calls, and usage tracking work through the OpenAI-compatible transport.

Env-var auth

API key interpolated from QWENCLOUD_API_KEY. Safe to commit, safe to share. Inline fallback available.

Custom tools

Wan and HappyHorse registered as native opencode tools via @opencode-ai/plugin. AI calls them autonomously — no special commands needed.

Refreshable models

Run fetch-models.mjs to pull the live model catalog. Auto-filters non-chat families.

Smoke tested

Four live API checks — basic completion, SSE streaming, tool call, second model — via smoke-test.mjs. Run locally before your first session.

Validated config

validate.mjs checks JSON syntax and required provider fields. CI runs it on every push.
