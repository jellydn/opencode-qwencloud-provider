opencode provider

QwenCloud models for opencode

Access Qwen3.8, Qwen3.7, Qwen3.6, DeepSeek V4, and GLM‑5.2 through QwenCloud's OpenAI-compatible API. Zero dependencies, config-only — drop in a JSON file and start coding.

Get started View on GitHub 
// opencode.json
"provider": {
  "qwencloud": {
    "npm": "@ai-sdk/openai-compatible",
    "name": "QwenCloud",
    "options": {
      "baseURL": "https://token-plan...",
      "apiKey": "{env:QWENCLOUD_API_KEY}"
    },
    "models": { … }
  }
}

No build step. No runtime code. Just JSON.

Install

Two steps to get started

1

Copy the config

Drop opencode.json into your global or project config.

cp opencode.json ~/.config/opencode/
2

Set your API key

Export your Token Plan key — it never touches the config file.

QWENCLOUD_API_KEY=sk-... opencode

Why

Zero dependencies, pure config

No build step

opencode reads the JSON directly. No npm install, no compile, no runtime code.

Env-var auth

API key interpolated from environment. Safe to commit, safe to share.

OpenAI-compatible

Streaming, tool calls, and usage tracking work out of the box via the standard adapter.
